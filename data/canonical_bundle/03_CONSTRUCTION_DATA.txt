==SECTION:: 03.construction_modulus.r3 ==

The reconstruction congruence modulus at level r=3 is

\[
\mathrm{MOD} \;=\; 81 \;=\; 3^4.
\]

Definition of \(a_0(n,m)\) at r=3:
For each live row index \(n \in U_3\) (unit) or \(n \in O_3\) (full),
and each live column index \(m \in U_4\), the exponent \(a_0(n, m)\)
is the unique integer in \(\{1, 2, \ldots, 54\}\) such that

\[
2^{a_0(n,m)}\,m \;\equiv\; 3n + 1 \pmod{81}.
\]

Why \(P_3 = 54\):
The multiplicative order of 2 modulo 81 is exactly 54. Verification:
\(\phi(81) = 54\); a direct check gives \(2^{27} \equiv -1 \pmod{81}\)
and therefore \(2^{54} \equiv 1 \pmod{81}\). So 54 is the period of
the powers of 2 modulo 81, which is exactly the period structure
inherited by the canonical denominator \(1 - t^{54}\) and the
normalization \(c_3 = h^2 / (1 - t^{54})^2\)
[ref: 02.source.r3_parameters].

What MOD = 81 is NOT:
  - it is NOT a competing convention parameter
  - it is NOT a side-channel ID
  - it is NOT a denominator period parameter (the denominator period
    is \(P_3 = 54\), a derived quantity)
  - it is NOT independent canonical metadata; it is construction-
    internal to the source rule

Why this matters:
Witness rounds at r=3 must use the correct congruence and the correct
\(a_0\) range. A witness that uses a different modulus or a different
\(a_0\) range is reconstructing a different matrix and is a
sanity-gate failure. ChatGPT's arithmetic gate at Stage 4 of the
researcher-handoff workflow [ref: 08.protocol.researcher_handoff]
must verify both \(\mathrm{MOD} = 81\) and \(a_0 \in \{1, \ldots, 54\}\)
when the round involves r=3 matrix construction.

Cross-reference: this same congruence pattern lifts to higher levels
with \(\mathrm{MOD} = 3^{r+1}\) and \(P_r = \mathrm{ord}_{\mathrm{MOD}}(2)\);
recording is reserved for future canonical levels and is NOT a
Verified Fact at any level above r=3 in this canonical.

==SECTION:: 03.abstract.r2.unit_factorization ==

The compact characteristic-polynomial identity for the abstract unit
matrix.

  det(y*I - S(A,B,C,D,E)) = (y - L0) * (y - L1) * Q(y)^2

  L0 = A + B + 2*C + D + E
  L1 = A - B + 2*C - D - E

  Q(y) = y^2
       + (-2*A + 2*C) * y
       + (A^2 - 2*A*C - B^2 + B*D + B*E + C^2 - D^2 + D*E - E^2)

  perron_identification : after canonical substitution,
                          L0(t) = R_unit(t) [ref: 03.rowsum.r2.unit].

[STATUS: Verified Fact]

==SECTION:: 03.scaling.b_to_s_relations ==

Per-model relations linking abstract S-level factorizations to the
B-level characteristic polynomial:

  unit r=2 :
    p_B(x) = c^6 * (x/c - L0(t)) * (x/c - L1(t)) * Q_t(x/c)^2

  full r=2 :
    p_B(x) = c^9 * (x/c - L0(t)) * (x/c - L1(t))^2 * C_t(x/c)^2

  unit r=3 :
    p_B(x) = c_3^{18} * p_S(x / c_3)

  full r=3 :
    p_B(x) = c_3^{27} * p_S(x / c_3)

When L0(t) = R(t) is identified with the Perron row-sum:

  (x/c - R(t)) = (x - c * R(t)) / c = (x - rho) / c

confirming the Perron anchor at the B-level. Used as a sanity check
in every compact-charpoly closure.

==SECTION:: 03.reduction_rings ==

Reduction rings used across slice-specific witnesses.

  s = 0.50 :
    t = 2^{-1/2} = sqrt(2)/2
    minimal relation : 2*t^2 - 1 = 0
    ring             : Q[t]/(2*t^2 - 1)
    isomorphic to    : Q(sqrt(2))
    h                : 3^{-1/2} (involves sqrt(3))

  s = 0.55 :
    t = 2^{-11/20}
    minimal relation : 2048*t^{20} - 1 = 0
    ring             : Q[t]/(2048*t^{20} - 1)
    basis            : 1, t, t^2, ..., t^{19}
    h_for_unit       : h = 3^{-1/10}, h^{10} = 1/3 in Q
                       bounded fact: h^k not in Q(t) for k = 1..6

  s = 0.60 :
    t = 2^{-3/5}
    minimal relation : 8*t^5 - 1 = 0
    ring             : Q[t]/(8*t^5 - 1)
    basis            : 1, t, t^2, t^3, t^4
    h                : 3^{-1/5}, h^5 = 1/3 in Q
                       bounded fact: h^k not in Q(t) for k = 1..4

  parity_warning :
    At s = 0.55, the minimal relation is even in t; reductions
    preserve even-power content.
    At s = 0.60, the minimal relation is odd-degree in t; reductions
    can transfer even-power content to odd basis positions because
    t^6 = t/8, t^8 = t^3/8, etc. Parity audits MUST account for the
    minimal relation of the reduction ring.

  cross_ring_warning : artifacts in Q[t]/(2048*t^{20} - 1) and
                       Q[t]/(8*t^5 - 1) are NOT interchangeable.
                       A historical Patched failure copied between
                       them. [ref: 06.withdrawn.ledger]
