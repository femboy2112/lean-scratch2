==SECTION:: 06.contradiction_guards ==

Active contradiction guards. Any future claim matching a guard pattern
halts the round.

  guard.descent_to_Q_at_non_rational_slices :
    For B_2^unit(0.55), B_2^full(0.55), B_2^unit(0.60),
    B_2^full(0.60):
    Any claim p_B(x) in Q[x] OR "descent to Q succeeds" is
    Contradiction Detected.

  guard.full_050_charpoly_incompatibility :
    Any claim of rank drop, rank 7, exact zero modes,
    near-zero-as-exact-zero, singularity, or any characteristic
    polynomial for B_2^full(0.50) incompatible with
      (1/261121) * (x - 1) * (7x - 1)^2 * (73*x^3 - 63*x^2 + 15*x - 1)^2
    is Contradiction Detected.

  guard.shifted_doubled :
    Any treatment of doubled-package artifacts as direct Frozen
    Convention A outputs is Contradiction Detected.

  guard.routine_reopening :
    Any reopening of a closed per-slice or structural layer as
    routine work is Contradiction Detected unless explicitly scoped
    as audit.

  guard.cross_level_inference :
    Any inference from closed r=2 to r=3 (or vice versa) without
    independent level-specific witness is Contradiction Detected.

  guard.compact_to_descent :
    Any treatment of compact S-level factorization as descent to Q
    at the B-level is Contradiction Detected.

  guard.r3_matrix_provenance :
    Any closure that invokes a specific integer or symbolic r=3
    matrix object (e.g., a 27 x 27 integer matrix A claimed to be
    "the full r=3 matrix") MUST declare:
      - the canonical s-slice or non-slice integer-clearance the
        matrix corresponds to
      - the explicit relationship A = (scaling) * S(t) at that slice
        OR an explicit derivation of A from the source rule with a
        clearly stated specialization
      - all parameters used in the construction (P_3, denominator,
        canonical ordering, MOD if any, a_0 definition source)
    Closures lacking matrix-provenance declaration cannot be
    integrated as canonical at the named slice level. They can be
    integrated only as Verified Fact closures of the specific
    matrix object (e.g., "spectrum of this 27 x 27 integer matrix"),
    not as r=3 slice-level closures.

  guard.factor_to_eigenvalue :
    Any conversion of compact-factor multiplicity (Q(y)^2, C(y)^2,
    (y - L1)^2 in r=2 closures) into numerical eigenvalue
    multiplicity without separable-factors witness is
    Contradiction Detected.

  guard.unit_to_full_transfer :
    Any claim that the r=2 unit reversal-involution mechanism
    transfers to the r=2 full model is Contradiction Detected.
    [ref: 06.failed_candidates entry "full_reversal_simple"]

  guard.full_partition_uniqueness_overreach :
    Any claim of full-partition uniqueness beyond the unordered
    3+3+3 case as currently witnessed is Not Established and must
    not be claimed without separate witness.
    [ref: 04.structural.partition.uniqueness]

==SECTION:: 06.over_upgrade_patterns ==

Patterns that have produced over-upgrade errors historically.
Every arithmetic gate and proof-boundary judgment must scan for these.

  pattern.compact_implies_rational :
    Treating compact S-level factorization as descent to Q at the
    B-level. Counterexample: the four r=2 non-rational descent
    closures, all at slices with compact factorizations but
    p_B(x) not in Q[x].

  pattern.squared_factor_implies_repeated_eigenvalue :
    Converting compact-factor multiplicity into numerical eigenvalue
    multiplicity without separable-factors witness. Forbidden.

  pattern.repeated_decimal_implies_multiplicity :
    Interpreting repeated decimals in solver output as exact
    multiplicities. Forbidden.

  pattern.near_zero_implies_zero_mode :
    Interpreting near-zero numerical eigenvalues as exact zero modes.
    Forbidden.

  pattern.cross_ring_coefficient_transfer :
    Copying coefficient witnesses between distinct reduction rings.
    Q[t]/(2048*t^{20} - 1) and Q[t]/(8*t^5 - 1) are NOT
    interchangeable.

  pattern.scaling_exponent_mixup :
    Mixing c^6 (unit) with c^9 (full) in r=2, or c_3^{18} with
    c_3^{27} in r=3. Each model has its own scaling exponent.

  pattern.parity_universal :
    Applying the s=0.55 parity rule (no odd reduced powers) uniformly
    across all reduction rings. Forbidden; the s=0.60 ring has odd
    minimal relation and produces odd basis terms after reduction.

  pattern.h_k_universal :
    Claiming h^k not in Q(t) for any k >= 1. Forbidden; h^{10} = 1/3
    at s=0.55 and h^5 = 1/3 at s=0.60. Use the bounded statement.

  pattern.r2_to_r3_lift :
    Lifting r=2 mechanisms (reversal, equitable partition) to r=3
    without independent r=3 witness. Forbidden.

  pattern.charpoly_variable_implicit :
    Implicit charpoly variable handling in symbolic-algebra libraries
    (e.g., SymPy's M.charpoly() without explicit variable). Caused
    a false row-sum-factor failure historically. Always declare
    explicit variable: y = symbols('y'); cp = M.charpoly(y).

  pattern.hardcoded_artifact_as_computation :
    Code that prints precomputed values from string literals rather
    than computing them. Forbidden by [ref: 08.protocol.artifact_standard].
